# คู่มือการติดตั้ง — RalphGuard

> ระบบประเมินความเสี่ยงการระคายเคืองและความเป็นพิษของสารเคมีด้วยแบบจำลองคอมพิวเตอร์
> NSC 2026 หมวด 14

คู่มือนี้อธิบายการติดตั้งและรันระบบทั้งหมดบนเครื่องเดียวด้วย Docker Compose ตั้งแต่ศูนย์

---

## 1. ความต้องการของระบบ (Prerequisites)

| เครื่องมือ | เวอร์ชันขั้นต่ำ | ตรวจสอบด้วย |
|---|---|---|
| Git | ใดก็ได้ | `git --version` |
| Docker Desktop | รองรับ Compose v2 | `docker --version` และ `docker compose version` |
| Node.js | 20+ | `node --version` |
| RAM | แนะนำ ≥ 8 GB | — |

> **หมายเหตุ:** ไม่จำเป็นต้องติดตั้ง Python, RDKit หรือ PostgreSQL บนเครื่องโดยตรง — ทุกอย่างรันใน Docker

## 2. ดาวน์โหลดซอร์สโค้ด

```bash
git clone https://github.com/lurk-z/ralphguard.git
cd ralphguard
```

> branch หลักที่มีโค้ดล่าสุดคือ `main`

ตั้งค่า environment:

```bash
cp .env.example .env
```

เปิดไฟล์ `.env` แล้วแก้ `API_SECRET_KEY` ให้เป็นค่าสุ่มที่ยาวพอ (สำหรับ production)

## 3. เตรียมชุดข้อมูลและฝึกแบบจำลอง (สำคัญ — ทำก่อนรัน worker)

ไฟล์แบบจำลอง `.pkl` **ไม่ได้อยู่ใน Git** (ถูกกันไว้ด้วย `.gitignore`) จึงต้องสร้างเองก่อนใช้งานจริง

### 3.1 วางชุดข้อมูล

ดาวน์โหลด CSV 4 ไฟล์ (ดูแหล่งใน [`data/raw/README.md`](../../data/raw/README.md)) แล้ววางใน `data/raw/`:

```
data/raw/
├── skin_irritation.csv
├── eye_irritation.csv
├── llna_sensitization.csv
└── catmos_acute_toxicity.csv
```

แต่ละไฟล์ต้องมีคอลัมน์ขั้นต่ำ: `smiles, label` (label = 0/1)

### 3.2 ฝึกแบบจำลองใน Docker

> **ทำไมต้องใน Docker?** สภาพแวดล้อม Python รุ่นใหม่บางเครื่องยังไม่มี wheel ของ scikit-learn 1.5.2 / RDKit จึงให้รันสคริปต์ในคอนเทนเนอร์ worker (Python 3.11) ที่มีไลบรารีครบ

**Windows (CMD):**
```bat
docker compose run --rm --no-deps -w /work -v "%cd%:/work" worker python data_prep.py
```

**macOS / Linux:**
```bash
docker compose run --rm --no-deps -w /work -v "$(pwd):/work" worker python data_prep.py
```

เมื่อเสร็จจะได้ไฟล์ในโฟลเดอร์ `scientific/models/`:

```
scientific/models/
├── skin_model.pkl
├── eye_model.pkl
├── sens_model.pkl
├── acute_model.pkl
└── validation_report.json
```

## 4. รันระบบทั้งหมด

### 4.1 ฐานข้อมูล + Redis + Backend + Worker (ใน Docker)

```bash
docker compose up -d --build postgres redis backend worker
```

- `--build` จำเป็นในการรันครั้งแรก และทุกครั้งที่แก้ `requirements.txt`
- backend จะรัน `alembic upgrade head` อัตโนมัติตอนเริ่ม เพื่อสร้างตารางในฐานข้อมูล
- backend และ worker mount โฟลเดอร์ `./scientific/models` เข้าไปที่ `/models` (อ่านอย่างเดียว) ตามตัวแปร `MODELS_DIR`

ตรวจสอบว่า backend พร้อม:
```bash
docker logs ralphguard-backend --tail 30
```
ควรเห็น `alembic upgrade head` สำเร็จ และ Uvicorn รันที่พอร์ต 8000

### 4.2 Frontend

**ตัวเลือก A — รันใน Docker (พร้อมกับ service อื่น):**
```bash
docker compose up -d --build frontend
```

**ตัวเลือก B — รันแยกเพื่อพัฒนา (แนะนำสำหรับ dev):**
```bash
cd frontend
npm install
npm run dev
```

## 5. ตรวจสอบว่าติดตั้งสำเร็จ

เปิดเบราว์เซอร์:

| บริการ | URL | ควรเห็น |
|---|---|---|

## 6. คำสั่งที่ใช้บ่อย

```bash
# ดู log
docker logs ralphguard-backend --tail 50
docker logs ralphguard-worker  --tail 50

# rebuild หลังแก้โค้ด/requirements ของ backend หรือ worker
docker compose up -d --build backend worker

# ฝึกโมเดลใหม่ (หลังแก้ dataset)
docker compose run --rm --no-deps -w /work -v "%cd%:/work" worker python data_prep.py

# หยุดทั้งหมด
docker compose down

# ตรวจ type ของ frontend
cd frontend && npm run type-check
```

## 7. การแก้ปัญหาที่พบบ่อย (Troubleshooting)

| อาการ | สาเหตุ | วิธีแก้ |
|---|---|---|
| `ModuleNotFoundError: rdkit` ใน backend | image เก่ายังไม่มี rdkit หลังแก้ requirements | `docker compose up -d --build backend worker` |
| Frontend ขึ้น "Failed to fetch" | backend ยังไม่ขึ้น / crash | ดู log backend แล้ว rebuild |
| `alembic upgrade head` ล้ม เรื่อง type `assessment_status` | enum ถูกสร้างซ้ำ | migration ใช้ `create_type=False` + `.create(checkfirst=True)` แล้ว — ตรวจว่าใช้ migration ล่าสุด |
| หน้า `/models` ไม่มีค่าเมตริก | ยังไม่ได้ฝึกโมเดล (`validation_report.json` หาย) | รัน `data_prep.py` (ข้อ 3.2) |
| worker log เตือน `predictor incomplete` | `.pkl` บางตัวหาย | ตรวจว่าฝึกครบ 4 endpoint แล้ว |
| Port ชนกัน (3000/8000/5432/6379) | มี container อื่นใช้พอร์ตอยู่ | `docker compose down` แล้วตรวจ `docker ps` |

## 8. โครงสร้างพอร์ตและ service

| Service | Container | Port | หน้าที่ |
|---|---|---|---|
| postgres | ralphguard-postgres | 5432 | ฐานข้อมูล |

| backend | ralphguard-backend | 8000 | FastAPI |
| worker | ralphguard-worker | — | ประมวลผล QSAR |
| frontend | ralphguard-frontend | 3000 | Next.js |
